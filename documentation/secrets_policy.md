# Политика управления секретами в проекте VR Kayak

## Что считается секретом
- Пароли от баз данных
- API-ключи (GitHub, Docker Hub, облачных провайдеров)
- Приватные ключи SSH
- Токены доступа (Personal Access Tokens)
- Строки подключения, содержащие пароли

## Где хранить секреты
- **GitHub Secrets** – для CI/CD переменных
- **Переменные окружения** в Docker Compose (только для локальной разработки, не коммитить)
- **HashiCorp Vault** (в production) – рекомендуется

## Запрещённые практики
- Хранение секретов в коде, конфигурационных файлах, `.env` в репозитории
- Коммиты с секретами в историю (использовать Gitleaks для блокировки)

## Что делать при утечке
1. Немедленно отозвать скомпрометированный секрет
2. Сменить секрет на новый
3. Создать инцидент и провести постмортем
4. Очистить историю Git через BFG Repo-Cleaner (если секрет уже был закоммичен)

## Инструменты контроля
- Gitleaks (CI) – блокирует пайплайн при обнаружении секретов
- Pre-commit hook – блокирует коммит локально
